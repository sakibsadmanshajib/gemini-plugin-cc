# Code of Conduct

## Scope

This Code of Conduct applies to all project spaces — issues, pull
requests, discussions, code reviews, and any project-affiliated
communication channels.

## Our pledge

We commit to making participation in this project a welcoming and
respectful experience for everyone, regardless of age, body size,
disability, ethnicity, gender identity, level of experience,
nationality, personal appearance, race, religion, or sexual identity.

## Expected behavior

- Be respectful and constructive in technical disagreements.
  Disagree with the work, not the person.
- Welcome newcomers; assume good faith on first contact.
- Give credit clearly. If a contributor finds a bug, surfaces an
  insight, or wrote the patch, name them.
- Document the _why_ in code review feedback so it teaches as well
  as it corrects.
- Honor the time others spend on this project. Read the docs +
  search existing issues before opening a new one.

## Unacceptable behavior

- Personal attacks, harassment, or discriminatory jokes — public or
  private.
- Sustained disruption (off-topic threads in PRs, repeatedly
  reopening closed issues without new information, etc.).
- Doxxing, stalking, or sharing private contact details without
  consent.
- Sexualized language, imagery, or unwanted attention of a sexual
  nature.

## Reporting

**Conduct issues:** email `conduct@artagon.dev` with the GitHub
permalinks involved + a short description. Reports are read only by
the maintainers; we'll acknowledge within 3 business days and
respond with the action taken (or the reason no action is
warranted) within 14 days.

**Security issues:** see [SECURITY.md](./SECURITY.md). Don't conflate
the two channels — code conduct goes to one inbox, vulnerabilities
to another.

## Enforcement

Maintainers may at their discretion:

- Edit or remove comments, commits, code, issues, or other
  contributions that violate this Code of Conduct.
- Temporarily or permanently ban contributors for behaviors they
  judge inappropriate, threatening, offensive, or harmful.

We try to act proportionally — a reminder for a first-time slip, a
ban only when the behavior is severe or repeated. We'll explain the
reason in the relevant thread or via direct message.
